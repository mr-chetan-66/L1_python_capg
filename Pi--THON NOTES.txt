    def static_get_kind():
        # Hard-codes Base, not subclass aware

    def class_get_kind(cls):
        # Respects subclass override
        return cls.kind

class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount  = amount
        self.deficit = amount - balance
        message = (f'Cannot withdraw {amount}. '
                   f'Balance: {balance}, Deficit: {self.deficit}')
        super().__init__(message)
 
# Usage:
try:

except InsufficientFundsError as e:

2) List & Dict Comprehensions (with if and if…else)


Comprehensions are compact syntaxes to build containers:

List comprehension → [ ... ]
Set comprehension → { ... }
Dict comprehension → {key: value for ...}
Generator expression → ( ... ) (like what you saw above)

A) List Comprehensions
Basic:
nums = [1, 2, 3, 4, 5]
squares = [x * x for x in nums]          # [1, 4, 9, 16, 25]

With a trailing if (filter):
even_squares = [x * x for x in nums if x % 2 == 0]  # [4, 16]
The if at the end filters which items are included.

With if…else (conditional expression in the value position):

labels = ["even" if x % 2 == 0 else "odd" for x in nums]
# ["odd", "even", "odd", "even", "odd"]

The if…else is part of the expression, not a filter.
You can not combine both forms in the same spot; but you can use both like this:

Combining value if…else + trailing if (filter):

# Label only the positive numbers; skip non-positives
labels_pos = ["even" if x % 2 == 0 else "odd" for x in nums if x > 0]
``

Multiple if filters (AND behavior):


# Keep numbers that are even AND > 2
filtered = [x for x in nums if x % 2 == 0 if x > 2]   # [4]
# Equivalent to: [x for x in nums if (x % 2 == 0 and x > 2)]


B) Dict Comprehensions
Basic:

words = ["apple", "bee", "carrot", "do"]
lengths = {w: len(w) for w in words}
# {'apple': 5, 'bee': 3, 'carrot': 6, 'do': 2}

With a trailing if (filter):

long_words = {w: len(w) for w in words if len(w) > 3}
# {'apple': 5, 'carrot': 6}

With if…else for values:
parity = {x: ("even" if x % 2 == 0 else "odd") for x in range(5)}
# {0: 'even', 1: 'odd', 2: 'even', 3: 'odd', 4: 'even'}

With if…else for keys (less common, but valid):
mapped = {(x if x % 2 == 0 else -x): x*x for x in range(5)}
# {0: 0, -1: 1, 2: 4, -3: 9, 4: 16}

C) Generator Expressions vs List Comprehensions

List comprehension ([...]) builds the whole list in memory (O(n) space).
Generator expression ((...)) yields items lazily (O(1) extra space), perfect for streaming operations like sum(), any(), all(), min(), max().


# List comp: builds a list first (uses memory)
sum_of_squares = sum([x*x for x in range(10_000_000)])  # heavier

# Generator expr: streams values to sum (better memory profile)
sum_of_squares = sum(x*x for x in range(10_000_000))



How database connectivity & execution works (in brief)

Connect: sqlite3.connect(path) opens a database file (creates it if absent).
Cursor: con.cursor() prepares a cursor to execute SQL.
Execute: cursor.execute(sql, params) sends the query and parameters to the engine. The engine parses, plans, and executes the statement.
Fetch: For SELECT, use fetchone(), fetchall(), or iterate the cursor.
Transactions:

By default, sqlite3 starts a transaction when you execute DML (INSERT/UPDATE/DELETE) and waits for con.commit() to persist changes.
You can call con.rollback() on errors.


Close: Always close cursor and connection (or better, use context managers).

from datetime import datetime
from dateutil.relativedelta import relativedelta  # pip install python-dateutil

def convert(s: str) -> datetime:
    return datetime.strptime(s, "%d-%m-%Y")

def main():
    d1 = convert(input("Enter date 1 (DD-MM-YYYY): "))
    d2 = convert(input("Enter date 2 (DD-MM-YYYY): "))

    rd = relativedelta(max(d1, d2), min(d1, d2))
    # You can get total months as:
    total_months = rd.years * 12 + rd.months
    print("Whole calendar months between (ignoring leftover days):", total_months)
    print("Breakdown -> years:", rd.years, "months:", rd.months, "days:", rd.days)

cursor.rowcount — number of rows affected by the last DML statement; key exam topic.


For Oracle drivers (cx_Oracle / python-oracledb), cursor.rowcount returns the number of rows affected by the last DML operation:

INSERT → number of rows inserted
UPDATE → number of rows updated
DELETE → number of rows deleted

So INSERT definitely updates rowcount, including executemany().

def get_employees_by_filters(min_salary=None, dept_id=None):
    query  = "SELECT employee_id, first_name, salary FROM employees WHERE 1=1"
    params = {}

    if min_salary is not None:
        query += " AND salary >= :min_salary"
        params["min_salary"] = min_salary

    if dept_id is not None:
        query += " AND department_id = :dept_id"
        params["dept_id"] = dept_id

    query += " ORDER BY salary DESC"

    cursor.execute(query, params)
    return cursor.fetchall()
